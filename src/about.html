PZ Map Hierarchy

Project Zomboid organizes its map data in a three-level hierarchy:

LevelDescriptionSize (B42)
TileThe smallest unit — a single square on the map1 × 1
ChunkA group of tiles, stored as one data block8 × 8 tiles
CellA group of chunks, the top-level map division32 × 32 chunks = 256 × 256 tiles

Save files are organized by cell and chunk coordinates. The cell coordinate determines which directory the data lives in, and the chunk coordinate determines the individual file names within that directory.

B41 vs B42 Comparison

Build 42 changed the map grid dimensions from B41. This matters because cleanup scripts need the correct constants to calculate which files to delete.

B41B42
Cell size300 × 300 tiles256 × 256 tiles
Chunk size10 × 10 tiles8 × 8 tiles
Chunks per cell30 × 3032 × 32

The key difference: B42 uses powers of two (256, 8, 32) instead of B41's round-but-arbitrary numbers (300, 10, 30). This is why scripts written for B41 will calculate wrong chunk ranges for B42 saves.

Coordinate Calculation

To find which chunk files belong to a given cell, multiply the cell coordinate by the number of chunks per cell:

chunk_start = cell_coord × chunks_per_cell
chunk_end   = chunk_start + chunks_per_cell - 1

Worked example: Cell 6, 57

X (cell 6)Y (cell 57)
Chunk start6 × 32 = 19257 × 32 = 1824
Chunk end192 + 31 = 2231824 + 31 = 1855

So cell (6, 57) contains all chunks where X ranges from 192–223 and Y ranges from 1824–1855. Any file in the save directory with chunk coordinates in that range belongs to this cell.

Save File Structure

A PZ save directory contains several subdirectories. Each uses either cell or chunk coordinates in its file names:

DirectoryCoordinate typeFile pattern
map_*Chunkmap_192_1824.bin
chunkdata_*Chunkchunkdata_192_1824.bin
zpop_*Cellzpop_6_57.bin

For chunk-based directories, the cleanup script must expand the cell coordinate into the full range of chunk coordinates and delete every matching file. For cell-based directories, it can delete by cell coordinate directly.
